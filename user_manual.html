왕초보용 A to Z 설명서

AI 글쓰기 회사 하네스, 처음부터 끝까지 따라하기

이 설명서는 “터미널이 뭔지도 헷갈리는 사람”을 기준으로 썼습니다. 무엇을 Codex 채팅창에 말해야 하고, 무엇을 터미널에 입력해야 하는지부터 천천히 설명합니다.

가장 중요한 답부터 말하면, python main.py는 Codex 채팅창에 쓰는 말이 아닙니다. 이 명령은 터미널에 입력하는 실행 명령입니다.

0. 먼저 용어부터 정리하기

Codex 채팅창

지금 당신이 나에게 “설명서 새로 만들어줘”라고 말하는 곳입니다. 부탁, 질문, 수정 요청을 쓰는 곳입니다.

터미널

컴퓨터에 실행 명령을 입력하는 검은색 또는 파란색 창입니다. PowerShell, 명령 프롬프트, Codex 앱의 터미널이 여기에 해당합니다.

하네스

Codex에게 일을 시키기 위한 운영 프로그램입니다. 작업지시서를 만들고, 결과 파일이 생겼는지 확인합니다.

헷갈리기 쉬운 부분: python main.py는 “나에게 부탁하는 문장”이 아니라 “컴퓨터에게 프로그램을 실행하라고 시키는 명령”입니다.

그래서 Codex 채팅창이 아니라 터미널에 붙여넣어야 합니다.

1. 이 프로그램으로 무엇을 하게 되나

당신은 글쓰기 회사의 CEO입니다. 직접 원고를 처음부터 끝까지 쓰기보다, “이번 책은 어떤 장르인지”, “어떤 분위기인지”, “무엇을 고쳐야 하는지”를 지시합니다.

하네스는 그 지시를 tasks/current_task.md라는 작업지시서로 만들고, Codex가 읽을 수 있게 정리합니다. 그리고 결과물이 정해진 파일에 생겼는지 확인합니다.

역할 하는 일
당신 CEO입니다. 장르, 방향, 승인, 수정 요청을 결정합니다.
하네스 회사 운영 시스템입니다. 상태, 작업지시서, 실행, 검증, 로그를 관리합니다.
Codex 직원입니다. 작업지시서를 읽고 기획, 목차, 초안, 편집본, 검수본을 만듭니다.

2. 가장 먼저 해야 할 일: 터미널 열기

이 단계가 제일 중요합니다. 아래 둘 중 하나를 고르세요. 둘 다 같은 결과입니다. 편한 방법 하나만 쓰면 됩니다.

방법 A: Windows 파일 탐색기에서 터미널 열기

  1. 파일 탐색기를 엽니다. 키보드에서 Win + E를 누르면 파일 탐색기가 열립니다.
  2. 주소창에 프로젝트 폴더를 붙여넣습니다. 아래 경로를 복사해서 파일 탐색기 주소창에 붙여넣고 Enter를 누릅니다.
    프로젝트 폴더 경로 
    C:\Users\user\Desktop\publisher-agent
  3. 같은 주소창에 powershell을 입력합니다. 폴더 안으로 들어온 뒤, 파일 탐색기 주소창을 한 번 클릭하고 아래 단어를 입력한 다음 Enter를 누릅니다.
    주소창에 입력 
    powershell
  4. PowerShell 창이 열리면 성공입니다. 창 앞부분에 C:\Users\user\Desktop\publisher-agent 비슷한 경로가 보이면 제대로 열린 것입니다.

방법 B: Codex 앱 안의 터미널 사용하기

Codex 앱에 터미널 패널이 보인다면 그곳을 써도 됩니다. 단, 채팅 입력칸이 아니라 터미널 입력칸이어야 합니다.

구분법은 간단합니다. 채팅창은 나에게 말을 거는 곳이고, 터미널은 PS C:\... 또는 > 같은 표시 뒤에 명령을 입력하는 곳입니다.

3. 프로그램 실행하기

터미널을 열었다면 이제 아래 명령을 복사해서 터미널에 붙여넣고 Enter를 누릅니다.

터미널에 입력 
python main.py
다시 한 번 강조합니다. 이 명령은 Codex 채팅창이 아니라 터미널에 입력합니다.

정상 실행되면 이런 메뉴가 보입니다.

화면에 보이는 메뉴 예시
AI 글쓰기 회사 하네스 MVP

--- 메뉴 ---
1. 새 책 만들기
2. 기획 생성
3. 목차 생성
4. 챕터 초안 생성
5. 편집본 생성
6. 설정 검수
7. 최종본 생성
8. 현재 상태 보기
9. 실행 모드 변경: dummy / live
0. 종료
메뉴 선택:

4. 처음 실행했을 때 추천 순서

  1. 1을 입력해서 새 책을 만듭니다. 책 제목과 장르를 물어보면 원하는 대로 입력합니다. 예: 제목은 “달빛 편집부”, 장르는 “판타지”.
  2. 2를 입력해서 기획을 만듭니다. “잔잔한 SF, 20대 독자, 따뜻하지만 쓸쓸한 분위기”처럼 추가 요청을 적습니다.
  3. 3을 입력해서 목차를 만듭니다. 이전 단계에서 만들어진 기획 파일을 바탕으로 목차 작업지시서가 생깁니다.
  4. 4를 입력해서 1장 초안을 만듭니다. 초안 결과 파일은 projects/book_001/chapters/ch001_draft.md에 생깁니다.
  5. 5, 6, 7을 차례대로 진행합니다. 편집, 설정 검수, 최종본 생성 순서입니다.
처음에는 dummy 모드라서 진짜 원고 대신 샘플 결과가 만들어집니다. 먼저 흐름을 익히고, 나중에 live 모드로 바꾸세요.

5. dummy 모드와 live 모드

모드 뜻 언제 쓰나
dummy 연습 모드입니다. 실제 Codex를 부르지 않고 샘플 파일만 만듭니다. 처음 배울 때, 폴더와 파일 흐름을 확인할 때 씁니다.
live 실제 Codex CLI를 실행하는 모드입니다. Codex 로그인이 되어 있고, 진짜 산출물을 만들 준비가 됐을 때 씁니다.

live 모드 전에 확인할 것

터미널에 아래 두 명령을 하나씩 입력해서 Codex가 설치되어 있는지 확인합니다.

Codex 버전 확인 
codex --version
Codex 실행 도움말 확인 
codex exec --help

6. 메뉴가 만드는 파일

메뉴 무엇을 시키나 결과 파일
1. 새 책 만들기프로젝트 상태 초기화projects/book_001/project.json
2. 기획 생성작품 설정과 방향 정리projects/book_001/story_bible.md
3. 목차 생성전체 목차와 1장 구조 만들기projects/book_001/outline.md
4. 챕터 초안 생성1장 초안 작성projects/book_001/chapters/ch001_draft.md
5. 편집본 생성초안을 읽기 좋게 편집projects/book_001/chapters/ch001_edited.md
6. 설정 검수설정 오류, 시간선, 인물관계 확인projects/book_001/reviews/ch001_continuity.md
7. 최종본 생성검수 내용을 반영한 최종본 생성projects/book_001/chapters/ch001_final.md
8. 현재 상태 보기현재 단계와 산출물 확인터미널 화면에 출력
9. 실행 모드 변경dummy와 live 전환project.json의 mode 변경

7. 무엇을 Codex 채팅창에 말하고, 무엇을 터미널에 입력하나

하고 싶은 일 입력하는 곳 예시
프로그램 실행 터미널 python main.py
테스트 실행 터미널 python -m unittest tests.test_harness_mvp -v
설명서 수정 요청 Codex 채팅창 “설명서를 더 쉽게 바꿔줘”
기능 추가 요청 Codex 채팅창 “챕터 번호를 고를 수 있게 만들어줘”
프로그램 메뉴 선택 터미널 1, 2, 0 같은 숫자

8. 좋은 요청을 쓰는 법

메뉴에서 “CEO 추가 요청”을 물어보면 짧지만 구체적으로 쓰는 것이 좋습니다.

좋은 요청 예시 
잔잔한 SF. 20대 독자 대상. 우울하지만 따뜻한 분위기. 폭력 묘사는 피하고, 주인공은 신입 편집자로 해줘.

좋은 요청: 장르, 독자, 분위기, 금지사항, 원하는 소재가 들어 있습니다.

아쉬운 요청: “재밌게 써줘”처럼 기준이 너무 넓으면 결과도 흔들립니다.

9. 문제가 생겼을 때

“python을 찾을 수 없다”고 나올 때

아래 명령을 대신 입력해 보세요.

대체 실행 명령 
py main.py

현재 폴더가 아닌 곳에서 실행한 것 같을 때

먼저 프로젝트 폴더로 이동한 뒤 실행합니다.

폴더 이동 후 실행 
cd C:\Users\user\Desktop\publisher-agent
python main.py

결과 파일이 안 생길 때

하네스는 실패를 감지하면 tasks/correction_task.md라는 재시도 작업지시서를 만들 수 있습니다. 이 파일은 “원래 작업에서 빠진 출력 파일을 다시 만들라”는 지시서입니다.

10. 테스트하는 법

하네스가 기본 기능을 지키는지 확인하려면 터미널에 아래 명령을 입력합니다.

테스트 실행 
python -m unittest tests.test_harness_mvp -v

성공하면 마지막에 OK가 보입니다.

11. 오늘 기억할 것 세 가지

  • python main.py는 터미널에 입력한다. Codex 채팅창에 입력하는 것이 아니다.
  • 처음에는 dummy 모드로 연습한다. 파일 흐름을 익힌 뒤 live 모드로 바꾼다.
  • 결과 파일은 projects/book_001 안에 생긴다. 메뉴 번호마다 결과 파일 위치가 정해져 있다.
