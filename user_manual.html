CEO용 설명서

AI 글쓰기 회사 하네스 사용 설명서

이 문서는 사람인 당신이 이 로컬 글쓰기 회사를 어떻게 운영하면 되는지 설명합니다. 하네스는 원고를 직접 쓰는 프로그램이 아니라, Codex에게 일을 맡기고 결과 파일을 회수하는 회사 운영 시스템입니다.

1. 이 회사가 하는 일

당신은 CEO입니다. 장르, 방향, 취향, 수정 요청을 말합니다. 하네스는 그 말을 직원별 작업지시서로 바꾸고, Codex가 읽을 파일을 만들고, 결과물이 정해진 위치에 생겼는지 확인합니다.

당신: CEO

작품의 방향을 정하고 승인합니다. 원고를 직접 붙잡고 고치기보다 요청과 판단에 집중합니다.

하네스: 운영 시스템

상태 관리, 작업지시서 생성, 실행, 검증, 로그 기록을 맡습니다.

Codex: 작업 직원

작업지시서를 읽고 지정된 파일에 기획, 목차, 초안, 편집본, 검수본을 작성합니다.

2. 가장 먼저 할 일

python main.py

위 명령을 실행하면 메뉴가 열립니다. 처음에는 dummy 모드입니다. 이 모드는 실제 Codex를 호출하지 않고 샘플 파일만 만들어서 회사 운영 흐름이 잘 움직이는지 확인합니다.

처음에는 1. 새 책 만들기를 선택하고 제목과 장르를 입력하세요. 그 다음 추천 순서대로 2 -> 3 -> 4 -> 5 -> 6 -> 7을 진행하면 됩니다.

3. 전체 흐름

CEO 요청당신이 장르, 분위기, 수정 요청을 입력합니다.
상태 확인project.json에서 현재 단계를 읽습니다.
작업지시서tasks/current_task.md를 만듭니다.
Codex 실행dummy 또는 live 모드로 실행합니다.
결과 회수출력 파일 존재 여부를 확인하고 로그를 남깁니다.

4. 메뉴별 의미

메뉴 역할 결과 파일
1. 새 책 만들기프로젝트 초기화projects/book_001/project.json
2. 기획 생성Plannerprojects/book_001/story_bible.md
3. 목차 생성Outlinerprojects/book_001/outline.md
4. 챕터 초안 생성Writerprojects/book_001/chapters/ch001_draft.md
5. 편집본 생성Editorprojects/book_001/chapters/ch001_edited.md
6. 설정 검수Continuity Checkerprojects/book_001/reviews/ch001_continuity.md
7. 최종본 생성Finalizerprojects/book_001/chapters/ch001_final.md
8. 현재 상태 보기상태 확인현재 단계와 산출물 목록 출력
9. 실행 모드 변경dummy/live 전환project.json의 mode 변경

5. dummy 모드와 live 모드

dummy 모드

안전한 연습 모드입니다. 실제 Codex를 호출하지 않습니다. 하네스가 작업지시서를 만들고, 지정된 출력 위치에 샘플 결과 파일을 만듭니다.

live 모드

Codex CLI를 실제로 호출할 준비가 된 모드입니다. OpenAI API 키가 아니라, 로컬에서 로그인된 Codex OAuth 세션을 사용합니다. 

codex exec --skip-git-repo-check -C <project-root> "<task prompt>"
live 모드로 바꾸기 전에 터미널에서 codex --version과 codex exec --help가 정상 동작하는지 확인하세요.

6. 중요한 파일

파일 또는 폴더 뜻
main.pyCEO가 실행하는 메뉴 프로그램입니다.
harness/상태 관리, 작업지시서 생성, 실행, 검증, 로그를 맡는 엔진입니다.
prompts/직원별 역할 카드입니다.
tasks/current_task.mdCodex가 읽을 최신 작업지시서입니다.
projects/book_001/첫 번째 책의 상태와 산출물이 모이는 곳입니다.
logs/runs.jsonl실행할 때마다 생기는 로그입니다. Git에는 올리지 않습니다.

7. 운영 규칙

좋은 사용법: 한 번에 하나의 작업만 시키고, 결과 파일을 확인한 뒤 다음 단계로 넘어갑니다.

좋은 요청: “잔잔한 SF, 20대 독자, 우울하지만 따뜻한 분위기, 폭력 묘사는 피하기”처럼 방향을 명확히 씁니다.

피할 것: 출력 파일 경로를 무시하고 아무 곳에나 원고를 만들게 하는 요청은 회사 운영 흐름을 깨뜨립니다.

8. 문제 해결

메뉴가 안 열린다

python --version
python main.py

Codex live 실행이 안 된다

codex --version
codex exec --help

두 명령이 실패하면 먼저 Codex 로그인 또는 설치 상태를 확인해야 합니다.

결과 파일이 없다

하네스가 tasks/correction_task.md를 만들 수 있습니다. 그 파일은 “누락된 출력 파일을 다시 만들라”는 재시도 지시서입니다.

하네스가 기본 계약을 지키는지 확인하려면 아래 명령을 실행합니다.

python -m unittest tests.test_harness_mvp -v
